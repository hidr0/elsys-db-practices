MySQL SELECT- избира различни колони от една таблица, селектираме нещо от някоя таблица

MySQL WHERE - поставяме условие за нещо, което да бъде изпълнено. Например в SELECT ако имаме таблица с човек и сложим WHERE, само хората над 25 години, то това трябва да е в WHERE.

MySQL AND, OR, NOT - пишат се, за да съпоставим две условия, пишат се най-често в WHERE, когато искаме да имаме няколко условия

MySQL ORDER BY - подрежда колони от една таблица по възходящ/низходящ ред. По подразбиране са във възходящ ред.

MySQL INSERT INTO - вкарваме някакъв запис в таблицата ни. Пише се по следния начин: 
INSERT INTO Table(column1, column2, column3, ….) VALUES(value1, value2, value3, …)
Ако искаме всичко да вкараме в някоя таблица може да не пишем колоните и ще ги има във всички.

MySQL NULL Values - нулеви стойности, могат да бъдат ползвани за някой запис в таблица, освен ако не сме казали, че искаме да е NOT NULL

MySQL UPDATE - актуализира дадена колона/запис в таблицата. Можем така да променим нещо от нашата таблица.
MySQL DELETE - изтрива колона/запис от нашата таблица
MySQL LIMIT - лимитира колко редове можем да изкараме. В случая, ако имаме голям SELECT, то ако лимитираме до 2, то ще ни покаже първите 2 реда от резултата.

MySQL MIN and MAX - взима най-голямата и най-малката стойност от нещо, което сме подали

MySQL COUNT, AVG, SUM - винаги трябва да бъдат следвани от името на колоната. Те броят по дадени критерии или всички записи в нашата таблица.

MySQL LIKE - казваме по какъв начин искаме да изглежда или трябва да изглежда нещо. Може да бъде сложено при създаването на таблица или също в WHERE.

MySQL Wildcards - 

MySQL IN - ползва се за проверки. Например ако имаме таблица с хора и искаме да изкараме само хората на 15, 20, 27 години ще напишем WHERE Person.age IN (15, 20, 27). Така може без проверки с AND,OR,NOT, а направо ще проверим всичко.

MySQL BETWEEN - ползва се също за проверка на някой запис. Почти същото е като IN, но тук вече слагаме някакъв range, в който да гледаме за стойности. При него се включват първата и последната стойности също.

MySQL Aliases - ползва се, за да си преименуваме нещо от таблицата в SELECT. За по-лесно, ако колоната от таблицата ни е с дълго име може просто да напишем as _name_, или as “name”, за да променим как да използваме името или да принитираме името.

MySQL Joins - свързва две таблици. Съществуват 4 типа на свързване на таблици: LEFT, RIGHT, CROSS, INNER. Винаги трябва да бъдат последвани от ON, където посочваме по какъв критерий да са свързани двете таблици.

MySQL INNER JOIN - свързва две таблици изцяло.

MySQL LEFT JOIN - залепва за първата таблица, втората отляво.

MySQL RIGHT JOIN - залепва за първата таблица, втората отдясно.

MySQL CROSS JOIN - свързва две таблици на кръст.

MySQL Self Join - не сме го учили.

MySQL UNION - не сме го учили.

MySQL DISTINCT - показва само записите от таблицата, които не са еднакви. Тоест ако имаме 3 еднакви реда/записа, то ако има DISTINCT ще покажем само 1 ред/запис.

MySQL GROUP BY - групира колони/записи. Тоест можем да кажем при един SELECT по коя от колоните да ни бъдат групирани записите.

MySQL Database - база от данни в MySQL, в която се съдържат таблици с данни.

MySQL Create DB - създаваме база данни.

MySQL Drop DB - ако таблицата вече съществува ще пуснем тази база от данни в нашето SQL.

MySQL Create Table - създава таблица, която е част от дадена база от данни.

MySQL Drop Table - пуска таблица, ако вече съществува.

MySQL Alter Table - не сме го учили

MySQL Constraints - някаква специална дума, която се ползва при създаване на таблици, за да кажем нещо от таблицата ни да е по някакъв критерий.

MySQL Not Null - constraint, казваме стойностите от тази колона в таблицата да не може да бъдат нулеви стойности.

MySQL Unique - constraint, казваме стойностите от тази колона да бъдат уникални и да не може да се повтарят.

MySQL Primary Key - constraint, уникален ключ, който се съдържа във всяка таблица за инициализация на всеки ред/запис.

MySQL Foreign Key - constraint, уникален ключ, който служи за свързване на две таблици.

MySQL Check - constraint, прави проверка за някоя стойност на колона от нашата таблица по зададен от нас критерий.
MySQL Default - constraint, казваме, че някакъв ред/запис от нашата таблица ако не му се подаде стойност да бъде с тази стойност по подразбиране.

MySQL Auto Increment - автоматично да увеличава нещо, всеки път, когато се създава нов запис от него.
